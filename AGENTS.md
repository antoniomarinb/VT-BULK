# AGENTS.md ‚Äî VirusTotal Simple Bulk Scanner

## Project Summary
Single-file Python CLI tool that bulk-scans files against VirusTotal API.
For each file: queries VT by SHA256 hash for existing analysis. If no valid
analysis exists (404), uploads the file to VT for scanning. Results are
displayed and optionally saved as JSON in `./scans/`.

## Tech Stack
- Language: Python 3.10+ (f-strings with nested quotes used)
- Dependencies: `requests` (auto-installed if missing), `argparse` (stdlib)
- Single script: `vt-bulk.0.3.3.py` (379 lines)
- Active branch: `dev`

## Script Architecture (vt-bulk.0.3.3.py)

### Constants & Globals
- `API_SCAN_REQUESTS_PER_MINUTE=4` ‚Äî VT rate limit
- `API_REQUEST_TIMEOUT=10` ‚Äî Timeout in seconds for all API requests
- `QUEUE_RETRY_DELAY=2` ‚Äî Seconds between poll retries
- `VERBOSE`, `NO_JSON_DUMP`, `CHECK_QUOTA` ‚Äî Toggle flags
- Global Queues: `analysis_results_queue`, `files_need_scanning_queue`,
  `path_and_link_to_requested_analysis_queue`
- Global: `finished_requesting_scans` (bool, signals workers to stop)
- Global: `headers` (dict, VT API auth headers)

### Execution Flow
1. Load API key from `vt_api_key.txt` (format: `key:userid`)
2. `argumentHandler()` ‚Äî parse CLI args or launch simple TUI
3. `getFilesToScan(dir, ext)` ‚Äî walk directory, filter by extension
4. `getUserVerification(files)` ‚Äî show files, ask user confirmation
5. `multithread_launchProgram(files)` ‚Äî main orchestrator:
   - Starts `requestedAnalysisWorker` thread (processes upload queue)
   - Spawns thread per file calling `multithread_GetFileResults()`
   - Each file: GET by SHA256 hash ‚Üí 200=existing analysis, 404=queue for upload
   - After initial scan join: polls `path_and_link_to_requested_analysis_queue`
   - Prints summary: malicious / suspicious / undetected lists
6. `printAndSaveDailyAPIQuotaStats()` ‚Äî fetch & display quota

### Key Functions
| Function | Purpose |
|---|---|
| `multithread_GetFileResults(file_path)` | GET file by SHA256 from VT; 200‚Üísave result, 404‚Üíqueue upload |
| `requestedAnalysisWorker()` | Worker thread: pulls from `files_need_scanning_queue`, POSTs files to VT with rate limiting |
| `getQueuedScansResultsV2()` | Polls `path_and_link_to_requested_analysis_queue` until all analyses complete |
| `APIRateLimiter` | Class: enforces per-minute request limit via time queue |
| `getFileHash(path, algo)` | Returns hex digest (SHA256/SHA512/MD5) |
| `createAnalysisFile(jsondump, path)` | Writes JSON to `./scans/{name}-{hash}.analysis.json` |
| `APIHelper()` | Interactive API key setup if `vt_api_key.txt` missing |
| `argumentHandler()` | Parses args via argparse: `-e`, `-q`, `-u`, `-f` and positional `path`; falls back to TUI if no path |
| `LaunchSimpleTUI()` | Interactive fallback when no CLI args provided |

### API Endpoints Used
- `GET /api/v3/files/{sha256}` ‚Äî Check existing analysis
- `POST /api/v3/files` ‚Äî Upload file for scanning
- `GET /api/v3/analyses/{id}` ‚Äî Poll analysis status (via link from POST)
- `GET /api/v3/users/{id}/api_usage` ‚Äî Quota stats

## Conventions
- No classes except `APIRateLimiter`; everything else is functions + globals
- `input2()` wraps `input()` with "exit" support
- Verbosity controlled by `VERBOSE` global flag (set via `-q`/`--quiet`)
- JSON dumps controlled by `NO_JSON_DUMP` global flag
- Files named with pattern: `{basename}-{sha256}.analysis.json`

## Testing
- `tests/test_unit.py` ‚Äî 36 unit tests (hashing, filtering, argparse, API mock, rate limiter)
- `tests/test_integration.py` ‚Äî 5 integration tests (all cached, all uploaded, mixed flow, polling)
- All API calls mocked with `unittest.mock` (no real VT requests)

- Run: `python -m pytest tests/ -v`
